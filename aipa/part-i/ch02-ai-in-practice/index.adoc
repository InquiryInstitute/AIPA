= Chapter 2: AI in Practice

This chapter shifts from conceptual foundations to practice: how intelligent systems operate in production. We cover real-world deployment (stakeholders, feedback loops, context-dependence), data pipelines and knowledge flow (who produces and consumes data), model lifecycle (training, deployment, drift), and observability (logs, metrics, traces, accountability). We examine failure modes and brittleness, human-AI collaboration, and AI system audits. The audit tool—inventory, traceability, report generation—is the chapter's deliverable. It integrates into student-ai/ as a tool server for self-inspection, extending the agent's capacity to observe and govern itself. Labs 1–3 build the audit tool from inventory through trace schema to full pipeline. By chapter end, you will have an operational audit tool and a framework for assessing deployed AI systems.

==== Learning Objectives
After this chapter you will be able to:
* Describe deployment, data pipelines, model lifecycle, and observability for AI systems
* Identify failure modes, brittleness, and human-AI collaboration patterns
* Build and integrate an audit tool (inventory, trace schema, pipeline)

==== Course Learning Outcomes (CLOs)
By the end of this course, you will be able to:
* Describe deployment, data pipelines, model lifecycle, and observability for AI systems (PLO 4).
* Identify failure modes, brittleness, and human-AI collaboration patterns (PLO 4).
* Build and integrate an audit tool (inventory, trace schema, pipeline) (PLO 2).
