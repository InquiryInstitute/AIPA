= Chapter 10: Architectures of Intelligence

This chapter covers agent architectures: pipelines vs. loops, multi-agent systems, orchestration (tool client), API integration, monitoring and debugging, scaling, and the orchestrator. The orchestrator ties everything together—llm, memory, reasoning, search, audit, governance. It lives in student-ai/orchestration/. Labs 1–3 build the pipeline design, tool client, and full orchestrator with APIs and monitoring. By chapter end, the agent is a complete distributed system. Ch11 adds governance; Ch12 is the capstone deployment.

==== Learning Objectives
After this chapter you will be able to:
* Compare pipelines vs. loops and design multi-agent systems
* Implement orchestration (tool client) and API integration
* Add monitoring, debugging, and scaling to the agent stack
* Build the orchestrator integrating llm, memory, reasoning, search, audit, governance

==== Course Learning Outcomes (CLOs)
By the end of this course, you will be able to:
* Compare pipelines vs. loops and design multi-agent systems (PLO 2).
* Implement orchestration (tool client) and API integration (PLO 2).
* Add monitoring, debugging, and scaling to the agent stack (PLO 4).
* Build the orchestrator integrating llm, memory, reasoning, search, audit, governance (PLO 5).
