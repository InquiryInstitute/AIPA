==== Conceptual Core

The student-ai/ agent is generic—it has the tools and structure from the course. The capstone makes it specific. Customization points: prompts (system prompt, few-shot examples), tools (add domain-specific tools or data), workflows (orchestrator routing, when to call which tool). Add domain-specific tools: e.g., an API for your industry, a database connector, a custom search index. Add domain data: index relevant documents in the memory store, configure the knowledge graph. The generic agent becomes situated.

Integration with existing systems: the agent may need to call APIs, query databases, or interact with legacy systems. The agent lives in context—it is not standalone. Testing: unit tests (components work in isolation), integration tests (end-to-end flow), user acceptance tests (does it solve the problem?). The agent in context: general vs. situated. The course agent is general; the capstone agent is situated—designed for a specific problem, users, and environment. Customization as design: we do not just configure—we design the agent for its situation.

==== Technical Example

Extend the agent for the capstone problem. Identify gaps: does the problem require a tool the course does not provide? Add it. Configure prompts: system prompt for the domain, few-shot examples for common patterns. Configure orchestrator routing: when should the agent call memory vs. search vs. a custom tool? ==== Philosophical Reflection

General vs. situated: the generic agent becomes specific through deployment. The course gives you a framework; the capstone gives you a problem. The work of customization is the work of design. We do not just set parameters—we make choices about what the agent should do, how it should behave, what tools it needs. The agent is a situated artifact—it exists in a context, for a purpose. Customization is design: we design for the situation.

The agent in context: the agent does not exist in isolation. It connects to APIs, databases, users. It has constraints—rate limits, latency, cost. The capstone agent is embedded. Understanding that embedding is part of the work. The agent that works in the lab may fail in the field—different data, different users, different expectations. Customization and testing bridge that gap.
