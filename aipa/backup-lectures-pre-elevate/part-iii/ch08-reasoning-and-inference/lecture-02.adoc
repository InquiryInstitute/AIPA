==== Conceptual Core

Variables and quantifiers—how do we reason about "all" and "some"? Propositional logic (L1) has no variables; we cannot express "every human is mortal" compactly. First-order logic adds terms (constants, variables, functions), predicates, and quantifiers (∀, ∃). That expressiveness scales logic to rich domains (Russell & Norvig, <<RussellNorvig2020>>)—but we pay in tractability: FOL inference is undecidable in general. This lecture covers the syntax, unification, and resolution for FOL—and where the limits lie.

FOL: terms (constants, variables, functions), predicates (relations), quantifiers (∀, ∃). Express "all X are Y," "some X is Y." Unification: find substitution that makes terms identical. Resolution for FOL: skolemization (remove ∃), convert to clauses, resolve with unification. Limitations: FOL is undecidable in general—no algorithm that always halts with correct answer. Expressiveness vs. tractability: FOL is more expressive than propositional; it is also harder to reason with. What can logic capture? Formal domains (math, programs); informal domains require approximation. The reasoning tool (L7) will expose FOL-style inference where the agent needs guaranteed entailment; probabilistic and LLM-based reasoning (L3, L5) extend the agent when the domain is uncertain or informal.

==== Philosophical Reflection

Expressiveness vs. tractability: we want to express more; we pay in computational cost. What can logic capture? Formal relations, yes. Vagueness, context, uncertainty—logic struggles. FOL is a powerful tool within its scope. So we choose: use FOL (and the reasoning tool) when we need precise, verifiable inference; use probability or LLMs when the domain is uncertain or open-ended.

===== Key Points

* Tradeoff: expressiveness, tractability
* Logic has scope limits
