=== 11.2: Cost Control and Rate Limiting
:leveloffset: 2

A research team deploys an autonomous literature‑review agent that runs 10 000 LLM calls overnight. By morning the cloud bill shows a $2 500 charge—far beyond the $200 daily budget. The system keeps answering, the team receives no warning, and the API key is throttled by the provider. *Who pays? What stopped the runaway spend?* This is the everyday reality of economic governance for AI agents.

"Every API call has a cost."
— EU AI Act <<EU2024>>; NIST AI RMF <<NIST2023>> (economic governance)

*Governance is also economic—control spend and usage.*

==== Conceptual Core

**Token‑level pricing**  
Every LLM call consumes input and output tokens; each token carries a provider‑defined price. By logging `tokens_in`, `tokens_out`, and the model’s per‑token rate, we obtain a granular cost for each request.

**Aggregating spend**  
Costs are summed per user, per task, and per day. Daily and monthly aggregates expose spending patterns and trigger alerts before budgets are exhausted.

**Rate‑limit mechanics**  
A rate limiter caps requests (e.g., 10 req/min per API key). When the limit is breached, the system returns a `429 Too Many Requests` response, preventing infinite retry loops and protecting shared compute resources.

**Cost‑aware design**  
Agents can cache repeated answers, batch multiple queries, or fall back to cheaper models for low‑stakes tasks. These patterns reduce token consumption without sacrificing functionality.

**Audit logs & compliance**  
Immutable logs of every cost‑related decision (budget checks, rate‑limit rejections) provide a forensic trail for auditors and support post‑mortem analysis of overspend incidents.

**Cost‑based fairness metrics**  
Policymakers increasingly measure AI equity by per‑capita spend or budget‑to‑outcome ratios. Embedding such KPIs into governance dashboards makes fairness a first‑class concern.

===== Key Points

* Token‑level cost tracking (tokens, model pricing)  
* Spend aggregation per user / task / day  
* Rate limiting (requests per minute) to stop runaway loops  
* Budgets, alerts, and configurable hard/soft stops  
* Cost‑aware design patterns: cache, batch, cheaper models  
* Immutable audit logs for compliance  
* Cost‑based fairness metrics as governance KPIs  

==== Technical Example

Below is a minimal implementation sketch for cost tracking and rate limiting in a Python‑style middleware.

[source,python]
----
# Cost calculator
def token_cost(model, tokens_in, tokens_out):
    price_per_token = PRICING[model]          # e.g. {"gpt‑4": 0.03/1k}
    return (tokens_in + tokens_out) * price_per_token

# Middleware wrapper
def handle_request(user, model, prompt):
    cost = token_cost(model, len(prompt), estimate_output(prompt))
    if not budget_ok(user, cost):
        raise BudgetExceededError(user)
    if not rate_limiter.allow(user):
        raise RateLimitError(user)
    # Record immutable audit entry
    audit_log.record(user, model, cost, timestamp())
    return call_llm(model, prompt)
----
Key technical additions:

* **Budget check** – compares projected cost against a daily `$50` limit; triggers a 403 response when exceeded.  
* **Rate limiter** – token‑bucket algorithm allowing 10 req/min; returns `429` on violation.  
* **Back‑off & jitter** – when a limit is hit, the client waits a random interval (e.g., 100‑500 ms) before retrying, reducing thundering‑herd effects.

==== Philosophical Reflection

Economic governance is a form of access control: who can afford to run agents at scale determines who participates in the AI ecosystem. When budgets are tight, only well‑funded organisations can deploy sophisticated agents, widening the digital divide. Embedding cost‑based fairness metrics—such as per‑capita spend or budget‑to‑outcome ratios—aligns technical controls with policy goals articulated in the EU AI Act and NIST AI RMF.

Cost control also underpins sustainability. Unlimited token consumption is financially and environmentally untenable for providers and users alike. By experimenting with the governance simulator (Lab 1), students see how tighter limits curb capability while looser limits boost utility but raise risk of runaway spend.

===== Key Points

* Cost = economic access control  
* Sustainability of AI services  
* Cost‑based fairness metrics as governance levers (EU/NIST)
