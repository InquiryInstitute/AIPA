==== Conceptual Core

Cognitivism emerged in the mid‑twentieth century as a reaction to behaviorism. Where behaviorism refused to speculate about internal states, cognitivism proposed that the mind processes information—that cognition is computation over representations. This shift opened a research program that treated mental operations as manipulable symbols.

Newell and Simon’s **Physical Symbol System Hypothesis** gave the proposal a precise form: a physical symbol system has the *necessary and sufficient* means for intelligent action. In other words, any system that can create, modify, and combine symbolic structures can, in principle, exhibit general intelligence. Early empirical work on problem‑solving tasks and computer simulations was presented as support for this bold claim.

Information‑processing psychology modeled the mind as a pipeline or loop: sense, represent, process, act, store. Attention, memory, and reasoning were studied as computational processes that could be broken down into discrete stages. The metaphor was powerful—it unified perception, memory, and problem‑solving under one framework and inspired the design of early cognitive architectures.

Critics quickly pointed out limits. **Embodiment** argued that we think with bodies, in environments, and that cognition cannot be fully captured by internal symbols alone. **Situated cognition** emphasized that intelligent behavior emerges from the coupling of agent and world, not merely from internal representation. These challenges reminded us that the computational model is a *map* rather than the full *territory* of lived experience.

The first major challenge to the purely symbolic view came from the rise of **connectionist** models in the 1980s. Neural networks compute over distributed, sub‑symbolic representations and learn from data, offering a different account of cognition that sidesteps explicit symbol manipulation. This tension between symbolic and connectionist approaches continues to shape modern AI, from symbolic knowledge‑graph systems to deep‑learning transformers.

===== Key Points

* Cognitivism: mind as computation over representations
* Physical symbol system hypothesis (Newell & Simon): symbols sufficient for intelligence
* Information-processing psychology: sense, represent, process, act, store
* Limits of the computational metaphor: embodiment, situated cognition
* Embodiment: we think with bodies, in environments
* Situated cognition: intelligence from agent–world coupling, not just internal representation
* Korzybski: the map is not the territory—model vs. lived experience

==== Technical Example

Production systems—collections of if‑then rules that fire when conditions match working memory—were among the first computational models of cognition. ACT‑R and Soar are symbolic architectures that implement this idea at scale. They model how humans solve problems: retrieve relevant knowledge, apply rules, update state, repeat. The architecture is inspectable; we can trace which productions fired and why. This transparency is a virtue for cognitive modeling—we want to understand the mind—but it is also a constraint. Not all cognition may be rule‑like.

A production system for a knowledge graph explorer might look like: if the user asks for related entities, then traverse edges and return neighbors. The rule is explicit. A neural retrieval model, by contrast, learns from examples; we cannot inspect its “rules” in the same way. The tension—interpretability versus flexibility—mirrors the symbolic–connectionist divide. Your explorer will support both: explicit traversal (symbolic, transparent) and, later, learned retrieval (connectionist, flexible). The cognitive pipeline—input, represent, process, act, store—maps onto your system: the user's query is input, the graph is the representation, traversal or retrieval is process, the returned results are output, and the graph itself is the store.

===== Key Points

* Production systems: if‑then rules over working memory
* ACT‑R, Soar: symbolic cognitive architectures
* Transparency: we can trace which rules fired
* Knowledge graph traversal: explicit rules (if user asks X, traverse Y)
* Neural retrieval: learned, not inspectable
* Your explorer: explicit traversal + later learned retrieval
* Pipeline: input → represent → process → act → store

==== Philosophical Reflection

Does computation exhaust cognition? The question is not settled. If the mind is a kind of computer, then building computers that pass the Turing test would be building minds. If not—if there is something about embodiment, phenomenology, or social embeddedness that computation cannot capture—then AI will forever fall short of human cognition. We need not resolve the issue to build useful systems. The computational metaphor has been enormously productive; it has also blinded us to aspects of intelligence that do not fit the frame.

"What gets left out when we model mind as program?" Possibilities: qualia (what it feels like), embodiment (how the body shapes thought), context (how situation constrains interpretation), and the social (how meaning is co‑constructed). The knowledge graph is a representational artifact; it does not feel, it does not have a body, it does not share a lifeworld with its user. But it can support cognition—as a map supports navigation. The map is not the territory, but we need maps to navigate.

===== Key Points

* Does computation exhaust cognition? Unsettled
* What gets left out: qualia, embodiment, context, the social
* We need not resolve the issue to build useful systems
* Knowledge graph: representational artifact, supports cognition
* The map is not the territory—but we need maps to navigate
