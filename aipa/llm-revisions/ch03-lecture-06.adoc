==== Conceptual Core

Fully observable environments: the agent sees the true state. Partially observable: the agent receives observations that depend on the state but do not fully determine it. pass:q[\aimaterm{belief state}]s: instead of a single state, the agent maintains a set (or distribution) of possible states. Search operates over belief states—each "node" is a belief state, and actions/observations induce transitions. pass:q[\aimaterm{POMDP}]s (Partially Observable pass:q[\aimaterm{Markov decision process}]s) formalize this: states, actions, observations, transition model, observation model, reward. The belief state is a distribution over states; belief update uses Bayes' rule: given action, observation, and prior belief, compute posterior belief.

Sensor models: how observations depend on state. Noisy sensors introduce uncertainty—we may mishear, missee. Belief update: b'(s') ∝ O(o|s') Σ_s T(s'|s,a) b(s). The cost of partial observability: the belief space is large (continuous, high‑dimensional); optimal solutions are intractable for big POMDPs. Approximation: particle filters, point‑based value iteration, or heuristic search over beliefs.

===== Key Points

* Fully vs. partially observable
* Belief state: set or distribution over states
* POMDP: states, actions, observations, transition, observation model
* Belief update: Bayes' rule
* Sensor models: observations depend on state
* Cost: belief space large, intractable
* Approximation: particles, point‑based, heuristics

==== Technical Example

Simple POMDP: navigation with noisy sensors. Agent does not know exact location; observations (e.g., "see wall to north") narrow the belief. Belief update via Bayes: prior belief, action (move north), observation (see wall), posterior belief. Implement a discrete POMDP with a few states—e.g., 1D grid, agent moves, observes left/right walls with noise. The search engine (Lab 3) may need to handle uncertain queries—e.g., user intent is ambiguous, or retrieval returns a distribution over relevant documents. Belief‑state reasoning is a bridge to probabilistic methods (Chapter 8).

[source,python]
----
# Pseudo‑code for belief update loop
b = initial_belief()                     # distribution over positions
while not terminated:
    a = policy(b)                         # choose action based on belief
    b_pred = T @ b                        # predict next belief (state transition)
    o = sense()                           # noisy observation
    b = O[o] * b_pred                     # element‑wise multiply by observation model
    b = b / b.sum()                       # normalize
----

==== Philosophical Reflection

Epistemic uncertainty in search: we do not know the state. Ignorance is not simply absence of knowledge—it has structure. We have a belief (distribution); we take an action; we get an observation; we update. The belief state encodes what we know and what we do not. The cost of partial observability: we must reason over beliefs, not states. The space explodes. In practice, we approximate—particles, point‑based methods, or we assume full observability and hope. When the assumption fails, the agent may act on wrong beliefs. Designing for partial observability is designing for uncertainty.

> “All models are wrong, but some are useful.”  
> — George Box, *Science and Statistics*  

Designing for hidden states raises ethical questions: an autonomous system that mis‑estimates its environment can cause harm. Moreover, when humans remain in the loop, they must understand the uncertainty conveyed by the system to make safe decisions.

===== Key Points

* Epistemic uncertainty: we don't know the state
* Belief encodes what we know and what we don't
* Cost: reasoning over beliefs, space explodes
* Assume full observability at your peril
* Ethical risk of hidden states influencing actions
* Human‑in‑the‑loop uncertainty handling
