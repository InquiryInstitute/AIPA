=== 3.1: Problem Spaces and State Representation
:leveloffset: 2

*You are dropped into a maze that only shows walls—no doors, no traps, no exit sign.

*The problem space is not the problem—it is a representation that enables search.*

==== Conceptual Core

In plain language: you have a set of possible situations, a starting situation, a goal, and actions that change one situation into another; search is finding a sequence of actions from start to goal. Formally:

[.formal-definition]
****
*Formal Definition (State space)*
A pass:q[\aimaterm{state space}] is a tuple (S, s₀, G, A, succ) where S is the set of states, s₀ is the initial state, G is the goal (or goal test), A is the set of actions, and succ: S × A → S is the pass:q[\aimaterm{transition model}]. pass:q[\aimaterm{Search}] is traversal of the state space from s₀ to a goal. See AIMA <<RussellNorvig2020>>.
****

Search algorithms operate on representations. We formulate problems as state spaces: a set of states, an initial state, a goal state (or goal test), and actions that transform states. The successor function defines the transition model—given a state and an action, what state results? The problem space is a graph: states are nodes; actions are edges. Search is traversal from initial to goal. The representation is not the problem itself; it is a reduction. Derrida reminds us that every representation excludes. What we include in the state—and what we omit—shapes what search can find.

State representation is an epistemic choice. The 8‑puzzle: we might represent only tile positions (sufficient for solvability) or include move count (for cost‑sensitive search). Navigation: we might use discrete grid cells or continuous coordinates. What to abstract away? Physics, psychology, social context—we reduce the world to what the search algorithm needs. Completeness (does search find a solution if one exists?) and optimality (does it find the best solution?) depend on the representation and the search strategy. The choice of representation is prior to the choice of algorithm.

===== Key Points

* State space: states, initial state, goal, actions
* Successor function: state + action → next state
* Problem space = graph; search = traversal
* Representation is reduction—what we include/omit matters
* Completeness: finds solution if exists
* Optimality: finds best solution
* Representation choice is epistemic—Derrida: every representation excludes
* Wrong abstraction → algorithm finds "solution" that fails in practice

==== Technical Example

Having formalized a state space, let’s see how it looks in a classic puzzle.  

Represent the 8‑puzzle: state = tuple of 9 tile positions (0 = empty). Actions: move empty up, down, left, right. State equality: same tuple. Successor generation: for each action, compute new tuple if legal. The representation is minimal—we need only positions to define the transition model. For navigation: state = (x, y) grid cell; actions = move N/S/E/W; goal = reach target cell. Implement state equality (same coordinates) and successor generation (neighbors, avoiding obstacles). The same search algorithms—BFS, DFS, A*—apply once the representation is fixed. Lab 1 asks you to formalize 2–3 domains as state spaces. Choose representations that balance expressiveness and tractability.

===== Key Points

* 8‑puzzle: state = tile positions; actions = move empty
* Navigation: state = (x,y); actions = N/S/E/W
* State equality and successor generation—implement once, reuse algorithms
* State hashing: use a hash table or set to store visited states efficiently
* Cost function: optional edge weights enable A* to prefer cheaper moves
* Duplicate detection: avoid re‑expanding already visited states

==== Philosophical Reflection

Abstraction is an epistemic choice. When we reduce a problem to states, we decide what matters. The 8‑puzzle state ignores the color of the tiles, the material of the board, the mood of the player. Those exclusions enable search—the algorithm could not operate on the full richness of the world. But what is lost? Sometimes the abstraction is wrong: we model the wrong variables, and search finds a "solution" that fails in practice. Sometimes the abstraction is right for the task but wrong for the context: the puzzle is solvable, but the human cannot execute the plan. Representation is the first act of search. Get it wrong, and no algorithm can save you. Borges's garden of forking paths (<<Borges1941>>) is a literary image of the state space we search—branching futures, each path a possibility.

===== Key Points

* Abstraction: what matters, what we exclude
* Wrong abstraction → solution fails in practice
* Representation is the first act of search
* Ethical stakes: mis‑representation can bias outcomes or reinforce inequities
* Iterative refinement: representations can be revised after failed searches
* Domain expertise: who supplies the abstraction shapes the search space
* Borgesian metaphor: state space as a garden of forking paths
