==== Conceptual Core

How do we stop the model from memorizing the training set?  
When a model fits every training point it often captures noise rather than signal, leading to poor generalisation on new data.

Regularization addresses this by **penalising model complexity**.  
The penalty is added to the loss function so that, all else equal, simpler hypotheses are preferred. In practice we use three common forms:

* **L1 (Lasso)** – penalty `Ω(w)=λ∑|w_i|`. The absolute‑value term drives many weights exactly to zero, yielding sparse solutions and implicit feature selection.  
* **L2 (Ridge)** – penalty `Ω(w)=λ∑w_i²`. Squaring the weights shrinks them toward zero but rarely eliminates them, preserving all features with reduced magnitude.  
* **Elastic Net** – a convex combination of L1 and L2, allowing a trade‑off between sparsity and shrinkage.

From a statistical perspective, regularisation is a **bias‑variance trade‑off**. Adding a penalty increases bias (the model is forced to be simpler) but reduces variance (the model becomes less sensitive to fluctuations in the training data). The optimal λ balances these forces.

**Cross‑validation** provides a data‑driven way to locate that balance. In *k‑fold* CV the data are split into *k* equally sized folds; we train on *k‑1* folds and validate on the remaining one, rotating the held‑out fold each round. The average validation score across folds yields a low‑variance estimate of out‑of‑sample performance. *Leave‑one‑out* CV is the extreme case where *k = n*.

Model selection then becomes a systematic search: we define a grid of hyper‑parameters (e.g., several λ values, choice of L1 vs. L2), run k‑fold CV for each candidate, and pick the configuration with the best average validation score. The `ml_trainer` API mirrors this workflow: it accepts a regularizer type and strength, runs CV internally, and returns the best‑scoring model together with the selected hyper‑parameters.

Together, regularization and cross‑validation give us a principled, data‑driven way to instantiate Occam’s razor in modern ML pipelines.

===== Key Points

* L1 (Lasso): sparsity, feature selection  
* L2 (Ridge): uniform shrinkage of weights  
* Elastic Net: blend of L1 + L2  
* Regularisation adds a penalty `Ω(w)` to the loss, trading bias for variance  
* k‑fold CV: split → train → validate → rotate → average score  
* Model selection: grid search + CV → pick λ & regularizer type  
* `ml_trainer` config: `regularizer`, `lambda`, `cv_folds`  

==== Technical Example

Below is a minimal Python‑style sketch showing how to configure `ml_trainer` with L1 regularisation and run a 5‑fold CV search over several λ values.

[source,python]
----
from ml_trainer import Trainer

# candidate regularisation strengths
lambdas = [0.001, 0.01, 0.1, 1.0]

best_score = -float('inf')
best_lambda = None
best_coef = None

for lam in lambdas:
    trainer = Trainer(
        regularizer='l1',          # choose L1 (Lasso)
        lambda_=lam,              # regularisation strength
        cv_folds=5                # 5‑fold cross‑validation
    )
    cv_score, coef = trainer.fit(X_train, y_train)
    print(f'λ={lam:.3f} → CV score={cv_score:.4f}')
    if cv_score > best_score:
        best_score, best_lambda, best_coef = cv_score, lam, coef

print(f'Best λ={best_lambda:.3f} with CV score={best_score:.4f}')
print('Resulting coefficients:', best_coef)
----

The table summarizes the CV results for the example data set:

[cols="1,1,1", options="header"]
|===
| λ (regularisation strength) | CV R² score | Coefficient pattern

| 0.001 | 0.78 | All non‑zero, modest shrinkage
| 0.01  | 0.81 | Few small coefficients, some near zero
| 0.1   | 0.79 | Many coefficients exactly zero (sparse)
| 1.0   | 0.72 | Over‑shrinkage, under‑fitting
|===

Interpreting the results, we see that λ = 0.01 yields the highest validation score while still producing a sparse model—exactly the balance we seek.

===== Key Points

* Code sketch: `Trainer(regularizer='l1', lambda_=lam, cv_folds=5)`  
* Grid‑search over λ, evaluate with CV, retain best model  
* Table of λ → CV score → coefficient pattern illustrates sparsity vs. shrinkage  
* Practical tip: start with a log‑spaced λ grid (e.g., 10⁻³ … 10¹)  

==== Philosophical Reflection

Occam’s razor in machine learning is the conviction that, *ceteris paribus*, the simpler model that explains the data is preferable. Regularisation makes this principle explicit: the choice of a penalty function encodes a **prior** over hypotheses. L1 imposes a prior that many weights are exactly zero; L2 imposes a Gaussian‑like prior that weights are small but non‑zero.

This prior‑driven view is a concrete manifestation of the **bias‑variance trade‑off**. By increasing bias (through stronger regularisation) we deliberately constrain the hypothesis space, which reduces variance and improves robustness to noisy training samples. The optimal bias‑variance balance is discovered empirically via cross‑validation, turning a philosophical stance into a measurable engineering process.

Beyond predictive performance, simplicity carries **interpretability** and **ethical** benefits. Sparse models (L1) are easier for humans to audit, facilitating transparency and fairness assessments. However, an overly simplistic prior can hide complex relationships that are essential for equitable outcomes. Practitioners must therefore calibrate regularisation strength with an awareness of domain‑specific stakes, not merely chase the highest CV score.

Together, regularisation, cross‑validation, and thoughtful prior selection embody a disciplined approach to model building—one that respects both statistical rigor and the broader societal implications of algorithmic decisions.

===== Key Points

* Regulariser = explicit prior (L1 → sparsity, L2 → small magnitudes)  
* Bias‑variance trade‑off formalises Occam’s razor in statistical terms  
* Cross‑validation empirically balances bias and variance  
* Sparsity enhances interpretability and supports ethical auditing  
* Over‑regularisation risks under‑fitting and obscuring genuine patterns  
* `ml_trainer` makes prior selection and calibration reproducible
