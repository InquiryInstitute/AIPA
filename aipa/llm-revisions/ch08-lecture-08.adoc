=== 8.8: Lab Integration — Inference Engine
:leveloffset: 2

Imagine an AI tutor that must decide whether a student's answer to a geometry problem is *provably* correct before offering feedback.  
If the system can only retrieve facts or generate natural‑language explanations, it may congratulate a wrong answer or flag a correct one as uncertain.  
Only a dedicated inference engine can *assert* premises, *query* relationships, and *prove* conclusions, turning raw retrieval into disciplined reasoning.

==== Conceptual Core

The cognitive triad that powers an autonomous learner consists of **memory** (store and retrieve facts), a **large language model** (produce fluent text), and a **reasoning engine** (perform inference).  
The reasoning engine supplies the missing *structured‑thought* layer: it transforms retrieved symbols into logical deductions, probabilistic assessments, or chain‑of‑thought prompts that the LLM can elaborate.

Why does inference matter for safety?  By requiring a proof before an action, the agent can avoid hallucinations and unintended behaviors.  In safety‑critical domains—education, medicine, robotics—this guardrail is essential, because a single erroneous inference can cascade into harmful outcomes.

From a software‑architecture perspective the engine lives in `student‑ai/reasoning/`.  Its public API consists of five entry points:

* `assert(statement)` – add a logical axiom.
* `query(pattern)` – retrieve matching facts.
* `prove(goal)` – attempt a formal proof (propositional or FOL).
* `infer_probability(query)` – compute a Bayesian estimate.
* `chain_of_thought(prompt)` – generate a CoT trace via the LLM.

These functions are called by the higher‑level agent during the ReAct loop (Chapter 9).  The engine therefore acts as a *submodule* that both consumes memory facts and produces prompts for the LLM, completing the triad.

===== Key Points

* Cognitive triad: memory + LLM + reasoning
* Inference engine provides structured thought and safety guardrails
* API entry points: `assert`, `query`, `prove`, `infer_probability`, `chain_of_thought`
* Located in `student‑ai/reasoning/` as a reusable submodule
* Connects memory (facts) ↔ reasoning (deduction) ↔ LLM (natural language)
* Enables the agent to prove goals before acting (see Chapter 9)
* Supports both deterministic logic and probabilistic inference
* Acts as the “thought” component inside the ReAct perception‑reasoning‑action cycle

==== Technical Example

Below is a minimal, runnable Python‑style snippet that demonstrates the full flow from assertion to LLM‑generated explanation.

[source,python]
----
from reasoning import Engine
from llm import LLM

engine = Engine()
llm = LLM()

# 1️⃣ Populate the knowledge base
engine.assert("forall x (Student(x) -> Learns(x))")
engine.assert("Alice is a Student")

# 2️⃣ Prove a goal
goal = "Learns(Alice)"
proof = engine.prove(goal)

# 3️⃣ Turn the proof into a chain‑of‑thought prompt
prompt = engine.chain_of_thought(f"Explain why {goal} holds.")
explanation = llm.generate(prompt)

print("Proof steps:", proof)
print("LLM explanation:", explanation)
----

**Setup** – The engine is instantiated and seeded with two axioms: a universal rule and a concrete fact about Alice.  

**Execution** – `prove` searches for a derivation of `Learns(Alice)`.  If successful, it returns a list of inference steps; otherwise it reports failure.  

**Verification** – The resulting proof is fed to `chain_of_thought`, which crafts a natural‑language prompt for the LLM.  The LLM then produces a human‑readable explanation that can be shown to the student.

The snippet illustrates the three‑stage pipeline that the agent will repeat inside the ReAct loop: **retrieve** → **reason** → **generate**.

===== Key Points

* Sample code shows full pipeline: assert → prove → CoT → LLM
* `prove` returns a structured proof object (list of inference steps)
* `chain_of_thought` bridges formal proof and natural language
* Error handling: `prove` raises `ProofError` if goal is unattainable
* Performance tip: cache intermediate results to avoid recomputation
* Unit‑test example included in the lab (assert‑prove‑explain test case)
* Expected output demonstrates both logical rigor and fluent explanation

==== Philosophical Reflection

Reasoning in autonomous agents raises a classic tension between **rule‑following** and **rule‑breaking**.  Wittgenstein reminds us that “to reason is to follow a rule—or to break one consciously.”  An agent that always obeys formal logic may be brittle, while one that occasionally overrides a proof to act on intuition risks unreliability.  The inference engine therefore embodies a *deliberate* balance: it enforces rules when they are available, yet defers to the LLM’s probabilistic judgment when the knowledge base is incomplete.

From an ethical standpoint, reasoning acts as a **moral guardrail**.  By demanding a proof before a consequential action—such as assigning a grade or recommending medical advice—the system makes its decision process transparent and auditable.  Nevertheless, probabilistic inference (`infer_probability`) acknowledges uncertainty, preventing the agent from presenting false certainty.

Looking ahead, hybrid formalisms that blend deductive logic, Bayesian reasoning, and generative language models promise richer cognition.  Yet limits remain: formal proof systems struggle with commonsense knowledge, while LLMs can hallucinate.  The future of AI reasoning lies in **orchestrating** these strengths, not in seeking a single universal formalism.

===== Key Points

* Rule‑following vs. rule‑breaking: when to trust formal proof vs. intuition
* Reasoning as an ethical guardrail: transparency and auditability
* Probabilistic inference acknowledges uncertainty, avoiding false certainty
* Limits of pure logic (commonsense gaps) and pure LLM (hallucination)
* Hybrid orchestration (logic + probability + language) as the path forward
* Wittgenstein’s insight applied to autonomous agents
* Structured thought as infrastructure for responsible action
